Title:   Killing Wasted Re-Renders with Production Hook Instrumentation
Stage:   Duck Stage 2
Day:     Day 1 — Thu, May 07, 2026
Time:    15:00 – 15:30 (CEST)
Location: Hall 3

About Session:
Modern React and React Native applications often suffer from invisible performance
problems: unnecessary re-renders, unstable dependency arrays, and effects that
execute far more often than developers expect.

In development, everything looks fine. In production, real users pay the price.

In this talk, I'll show how we instrumented React hooks at build time, measured
their behavior in production, and discovered that nearly half of our component
updates were unnecessary. By wrapping useEffect, useMemo, and render cycles with
lightweight compile-time transforms, we collected real-world data without relying
on DevTools or manual profiling.

You'll see how this instrumentation works under the hood, what surprising patterns
emerged from production telemetry, and how these insights helped us significantly
reduce wasted renders and improve real user experience.

Instead of guessing where performance issues come from, we'll learn how to observe
React from the inside — safely, systematically, and at scale.

SESSION 2

Title:   The New Frontend Stack: Humans, AI, and Prompts
Stage:   Duck Stage 2
Day:     Day 2 — Fri, May 08, 2026
Time:    09:30 – 10:00 (CEST)
Location: Hall 3

About Session:
Modern frontend development is rapidly changing with the rise of AI-assisted
coding tools. Developers are no longer the only ones writing code — large language
models are becoming active participants in the development workflow. This introduces
a new layer in the frontend stack: prompts. Instead of writing every line manually,
developers increasingly describe intent through prompts that generate UI components,
API integrations, and application logic.

But this shift raises important engineering questions: How do we design systems
when part of the code is generated by AI?

How do we debug non-deterministic code? What new role does the developer play in
this architecture? In this talk we explore the emerging frontend stack: Humans, AI,
and Prompts, examine how AI tools reshape development workflows, and discuss
practical patterns for integrating AI into production-grade engineering.

SESSION 3

Title:   AST + AI = Developer Co-Pilot 2.0
Stage:   Duck Stage 3
Day:     Day 2 — Fri, May 08, 2026
Time:    11:00 – 11:30 (CEST)
Location: Hall 3

About Session:
How code understanding and automation can transform your team's and clients'
productivity.

What if your codebase could improve itself?

In this talk, we'll explore how combining Abstract Syntax Trees (AST) with AI can
turn your development workflow into a self-evolving system. While AI tools like
Copilot help us write code faster, they still lack a deep understanding of
structure and context. ASTs, on the other hand, can precisely navigate, analyze,
and transform code — safely and automatically.

By merging these two forces, we can build intelligent automation that doesn't just
suggest improvements but actually applies them — from API migrations and performance
optimizations to automated instrumentation and observability hooks.

I'll share how our team uses AST-based transformations and AI-driven insights to
eliminate repetitive work, speed up code reviews, and let engineers focus on
solving real problems instead of chasing syntax.

You'll walk away with:
  1. A practical understanding of how ASTs can power AI-driven code automation
  2. Real examples of self-updating and self-healing code
  3. Strategies to integrate these ideas into your team's productivity toolkit
